# 車種カタログデータの扱い

このアプリでは、検索と出品補完のために `src/data/catalog.ts` の車種マスターを参照します。

## 現在の方針

- Goo-net 自動車カタログのような「メーカー → 車種 → 世代/型式/装備候補」の構造に合わせる。
- 詳細マスターは `src/data/catalog.ts`、1000車種以上の大量検索候補は `src/data/catalogBulk.ts` に分ける。
- Goo-net の掲載内容を丸ごと複製せず、商用リリース時は正規ライセンス済みデータ、自社作成データ、または提携APIに差し替える。
- 大量検索候補は NHTSA VPIC public vehicle model data を補助ソースとして生成しているため、日本市場向け表記や型式の精度は詳細マスター側で順次補強する。

## 取り込みフォーマット

外部データを取り込む場合は、次の JSON 形に正規化します。

```json
[
  {
    "name": "トヨタ",
    "country": "日本",
    "sourcePath": "/catalog/TOYOTA/",
    "models": [
      {
        "name": "ハリアー",
        "bodyTypes": ["SUV・クロカン"],
        "generationHints": ["60系", "80系"],
        "optionHints": ["JBL", "パノラマルーフ"]
      }
    ]
  }
]
```

## 本番で必要な確認

- データ提供元との利用許諾
- 再配布、キャッシュ、AI学習利用の可否
- 更新頻度と差分更新方法
- 車検証OCRや電子車検証データと車種マスターの名寄せルール
