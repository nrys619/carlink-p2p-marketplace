# Operations Runbook

## Daily Checks

- `/api/health` が `ok: true` を返す
- `/api/readiness` の checks がすべて `ok: true`
- 新規出品の本人確認、車検証情報、写真枚数、価格異常値を確認
- 通報、キャンセル、返金相談、名義変更遅延を確認

## Incident Types

### 入金後に車両引き渡しが進まない

1. チャットログと入金ステータスを確認
2. 売主へ期限付き通知
3. 期限超過で取引停止、返金可否を決済事業者へ確認
4. アカウント制限と証跡保存

### 名義変更が完了しない

1. 必要書類の不足を確認
2. 行政書士・買主・売主へ不足書類を通知
3. 自動車税、保険、駐車場証明、ナンバー変更の条件を確認
4. 一定期限後に運営介入

### 車両状態の相違

1. 出品写真、説明文、チャット、現車確認メモを確認
2. 修復歴、冠水、メーター改ざんなど重大な相違を分類
3. キャンセル、減額、取引継続の選択肢を提示
4. 悪質な場合は出品停止

## Admin Requirements

- 出品一覧、ユーザー一覧、取引一覧
- 本人確認ステータス変更
- 出品停止、チャット閲覧、証跡ダウンロード
- 返金/キャンセル依頼のステータス管理
- 古物台帳に必要な取引記録の出力
